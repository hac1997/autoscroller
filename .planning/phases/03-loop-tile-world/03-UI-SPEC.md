# Phase 3 -- UI Design Contract

> Visual and interaction contract for the loop + tile world phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Phaser 3 canvas rendering) |
| Preset | not applicable |
| Component library | none -- all UI is Phaser GameObjects (Text, Rectangle, Container, Graphics) |
| Icon library | none -- unicode glyphs and geometric shapes (inherited from Phase 1/2) |
| Font | System stack via Phaser: `"Inter, system-ui, Avenir, Helvetica, Arial, sans-serif"` (inherited from `style.css`) |

**Context:** Inherits all conventions from Phase 1 and Phase 2 UI-SPECs. Canvas is 800x600. All in-game UI is Phaser GameObjects. This phase introduces 8 new UI surfaces: Loop World HUD, Planning Phase Overlay, Tile Placement Grid, Shop Scene (extended), Rest Site Scene, Event Scene, Treasure Scene, and Boss Exit Choice Scene.

**Phase 3 usage mapping:**
- **Label (14px):** Tile point cost values, tile inventory counts, adjacency bonus descriptions, difficulty multiplier, loop progress fraction
- **Body (16px):** Tile names in inventory/grid, event narrative text, rest site option descriptions, treasure item names, shop item labels
- **Heading (24px):** Scene titles ("Planning Phase", "Rest Site", "Event", "Treasure", "Boss Defeated"), button labels, tile point balance
- **Display (32px):** "LOOP {N}" announcement, "BOSS!" encounter text, "EXIT RUN" / "CONTINUE" choice labels, loop completion celebration

## Component Inventory (Phaser GameObjects)

### Tile Visual (reusable across world and planning views)
- Rectangle: 80px wide x 80px tall, fill color from tile type config
- Stroke: 2px, white when hovered, accent when selected for placement
- Alternating shade for basic tiles: even index `0x666666`, odd index `0x888888`
- Tile icon: unicode glyph centered in tile, Body (16px)
  - Basic: none (empty path)
  - Forest: "T" (tree glyph placeholder)
  - Graveyard: "+" (cross glyph)
  - Swamp: "~" (wave glyph)
  - Shop: diamond glyph (inherited from HUD)
  - Rest: "R" (blue text)
  - Event: "?" (purple text)
  - Treasure: "!" (orange text)
  - Boss: skull unicode or "B" (red text)
- Adjacency synergy indicator: 4px colored strip on the edge facing the synergy neighbor (synergy color `#ff00ff`)
- Hover state: stroke brightens to white, tooltip appears above tile (200ms delay)

### Tile Tooltip (appears on hover over any tile)
- Rectangle: auto-width (min 120px) x 60px at `0x222222`, alpha 0.9
- Positioned 8px above hovered tile, centered horizontally
- Tile name: Body (16px), white, top line
- Tile effect: Label (14px), muted, second line (e.g., "Spawns forest enemies", "Recover HP")
- Adjacency bonus: Label (14px), synergy color, third line (if active, e.g., "+10% gold drops")

### Loop World View (Game Scene - main gameplay)
- Side-view autoscroll: tiles rendered as 80x80 rectangles at y=450 (ground level), inherited from MapManager
- Hero sprite at x=100 (screen-relative via camera follow), y=410 (above tiles)
- Camera follows hero with deadzone (inherited pattern: `startFollow(player, true, 0.1, 0.1)`)
- Tiles generate ahead and clean up behind (existing expandTrack pattern, 10 tiles buffer)
- Loop seam: no visual break between last tile and first tile re-appearing
- Background: dominant color `#1a1a2e`

### Loop World HUD (Game Scene - top area, extends Phase 2 HUD)
- Container at y=0, full width, setScrollFactor(0), setDepth(100)
- Background panel: Rectangle 260px wide x 90px tall at `0x000000`, alpha 0.5, origin (0,0) at (10, 10)
- **Row 1 (y=20):** Gold icon (diamond, accent color, 20px) + gold amount (Body 16px, accent)
- **Row 2 (y=42):** Loop counter "Loop {N}" (Body 16px, white) + difficulty "x{mult}" (Label 14px, muted, right-aligned)
- **Row 3 (y=62):** HP bar (160px wide, 12px tall, semantic color) + HP text (Label 14px)
- **Right-side info (x=600):**
  - Tile points: tile point icon + "{N} TP" (Body 16px, tile points color)
  - Meta-loot: star icon + "{N}" (Label 14px, meta-loot color)

### Planning Phase Overlay (Between loops)

- Title: "Planning Phase" at Heading (24px bold), white, centered at y=30

  - Occupied tiles: filled with tile color, non-interactive (already placed)

## Interaction Patterns

### Loop Traversal (no player input during autoscroll)
1. Hero walks right automatically at constant speed (base ~60px/s, scaling per loop)
2. Camera follows with deadzone (inherited pattern)
3. When hero reaches a tile, EventBus dispatches `tile-entered` with tile data
4. Combat tiles trigger transition to CombatScene (Phase 2)
5. Special tiles (shop, rest, event, treasure) trigger their respective overlay scenes
6. Boss tile triggers CombatScene, then BossExitChoice on victory
7. After last tile, loop seamlessly wraps; `loop-completed` event fires
8. Loop completion celebration plays, then PlanningPhase overlay opens

### Planning Phase Flow
1. Planning overlay appears after loop completion celebration
2. Player sees miniature loop layout with all tiles
3. Player selects a tile type from inventory (click tile card)
4. Player clicks an empty slot in the loop layout to place it
5. Placed tiles appear immediately with 200ms scale-in animation
6. Adjacency synergy indicators update in real-time as tiles are placed
7. Player can place multiple tiles per planning phase (limited by inventory/points)
8. Player clicks "Start Loop" to begin next loop
9. Planning overlay fades out (300ms), hero resumes walking

### Tile Purchase in Planning Phase
1. Player clicks a purchasable tile card (not from drops)
2. If sufficient tile points: tile is "selected" (accent highlight), points deducted on placement
3. If insufficient tile points: card is dimmed, "Not enough tile points." toast appears (Label 14px, destructive, bottom-center, 1.5s fade)
4. Dropped tiles (free) show "x{count}" and deduct from inventory on placement

### Rest Site Interaction
1. Three option cards appear with 300ms staggered entrance
2. Player clicks an option to select (accent highlight, others dim)
3. "Choose" button appears below
4. On confirm: cards fade out (300ms), result text fades in (300ms)
5. Auto-continue after 2s or on click anywhere

### Event Interaction
1. Narrative text appears with typewriter effect (30 chars/second)
2. Choice buttons slide in from right (200ms staggered)
3. Player clicks a choice
4. Other choices fade out (300ms), selected choice gets accent stroke
5. Outcome text appears below narrative (300ms fade in)
6. "Continue" button appears at bottom

### Treasure Interaction
1. Loot items appear with staggered entrance (200ms per item, slide right + fade in)
2. Player reviews all items
3. "Take All" button appears after last item animates in
4. On click: items slide left + fade out (150ms staggered), overlay closes

### Boss Exit Choice
1. "Boss Defeated!" title appears with scale-in animation (Display, accent)
2. Two option panels appear simultaneously
3. Player clicks a panel to select (accent stroke, other dims)
4. Confirm button appears below selected panel
5. On "Exit Run": transition to run summary screen (extends Phase 2 Death Screen format but with "Run Complete" title and safe exit colors)
6. On "Continue": overlay closes, loop grows by +3 tiles, hero resumes

### Scene Transitions (inherits Phase 1/2 pattern)
- All scenes read from RunState via `getRun()` in `create()`
- No data passing via `scene.launch('SceneName', { data })`
- Special tile scenes are overlay scenes (launched on top of Game scene, not replacing it)
- Planning phase overlay pauses the Game scene update loop

## Phase-Specific Notes

Phase 3 transforms the game from isolated combat encounters into a connected world. The player's strategic identity shifts from "deck architect" (Phase 2) to "world architect" -- placing tiles that define their loop is the new primary input.

Key design principles for this phase:
1. **Planning Phase is the new strategic hub.** The miniature loop layout with tile placement is the equivalent of Phase 2's deck editor. It must feel satisfying and show consequences (adjacency synergies) immediately.
2. **Autoscroll is hands-off by design.** Like combat, the hero traverses tiles automatically. The player watches their decisions play out. No movement input.
3. **Tile colors are the visual language.** Each tile type has a distinct, consistent color (inherited from existing TileTypes.ts config). Players learn tile types by color before reading labels.
4. **Boss exit is the climax decision.** The two-panel layout with green (safe) vs red (risk) must make the stakes viscerally clear. Reward preview numbers reinforce the choice.
5. **Special tiles are brief overlays, not full scene transitions.** Shop/rest/event/treasure pop over the world scene to maintain spatial continuity. The player always returns to the loop.
6. **Adjacency synergies must be visible in the planning phase.** Colored edge strips on synergizing tiles let the player see combos forming as they place tiles.
