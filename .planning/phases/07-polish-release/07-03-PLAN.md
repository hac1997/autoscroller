**For each full scene, apply this pattern:**

1. Add a `private transitioning = false;` field to the class

2. In `create()`, add at the START (before other content):
```typescript
this.transitioning = false;
this.cameras.main.fadeIn(LAYOUT.fadeDuration, 0, 0, 0);
```

3. Create a helper method:
```typescript
private fadeToScene(sceneKey: string, data?: any): void {
  if (this.transitioning) return;
  this.transitioning = true;
  this.cameras.main.fadeOut(LAYOUT.fadeDuration, 0, 0, 0);
  this.cameras.main.once('camerafadeoutcomplete', () => {
    this.scene.start(sceneKey, data);
  });
}
```

4. Replace ALL `this.scene.start('SceneName', data)` calls with `this.fadeToScene('SceneName', data)` EXCEPT:
   - `this.scene.start` calls inside overlay scenes (they don't fade)
   - `this.scene.launch` calls (these are overlay launches, keep as-is)
   - `this.scene.stop` calls (keep as-is)
   - `this.scene.resume` calls (keep as-is)

**Button standardization:**
Replace the 7-line button creation pattern:
```typescript
const btn = this.add.text(x, y, 'Label', { fontSize: '24px', fontStyle: 'bold', color: '#ffd700' })
  .setOrigin(0.5).setInteractive({ useHandCursor: true });
btn.on('pointerover', () => btn.setColor('#ffffff'));
btn.on('pointerout', () => btn.setColor('#ffd700'));
btn.on('pointerdown', () => action());
```
With:
```typescript
const btn = createButton(this, x, y, 'Label', () => action(), 'primary');
```

Use 'primary' for main action buttons (24px bold), 'secondary' for smaller/less important buttons (16px).

**Important:** Do NOT replace colors that are semantically different from the standard palette. For example:
- Enemy health bar colors that are intentionally different
- Tile type colors that encode game meaning (these should stay as-is in GameScene/TileVisual)
- Temporary flash/damage indicator colors
