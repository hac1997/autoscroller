**Per user decision: "Tweened number counters on gold/HP/material changes in HUD (count up/down over 300ms rather than instant)"**

**1. Update `src/ui/LoopHUD.ts`:**

Add tweened counter logic to the `update()` method for gold, HP, and material displays. For each numeric display that currently does instant `setText()`:

- Track the previous displayed value in a private field (e.g. `private displayedGold: number = 0`)
- When the actual value differs from displayedGold, start a tween counter:
```typescript
this.scene.tweens.addCounter({
  from: this.displayedGold,
  to: newGold,
  duration: 300,
  onUpdate: (tween) => {
    const val = Math.round(tween.getValue());
    this.goldText.setText(String(val));
  },
  onComplete: () => {
    this.displayedGold = newGold;
  }
});
```
- Apply this pattern to: `goldText` (gold amount), `hpText`/`hpBar` (HP value and bar width), `materialsText` (material count)
- Guard against multiple concurrent tweens on the same field: before starting a new tween, stop any existing one for that field. Store tween references: `private goldTween?: Phaser.Tweens.Tween`
- For HP bar width, tween the bar's `displayWidth` alongside the text counter so the bar smoothly shrinks/grows

**2. Update `src/ui/CombatHUD.ts`:**

Apply the same tweened counter pattern to:
- `hpText` and `hpBar` (hero HP)
- `staminaText` and `staminaBar` (hero stamina)
- `manaText` and `manaBar` (hero mana)
- `enemyHpText` and `enemyHpBar` (enemy HP)

For each, track `displayedHeroHp`, `displayedStamina`, `displayedMana`, `displayedEnemyHp` and use `this.scene.tweens.addCounter()` with 300ms duration when the value changes.

**Important:** The tween uses the scene's tween manager (`this.scene.tweens` for LoopHUD which extends Container, and `this.scene.tweens` for CombatHUD which stores `this.scene`). Both already have access to the scene reference.
