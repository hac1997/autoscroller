Read the existing ShopScene structure first to understand the tab system. Then:

- Add a new tab button labeled "Upgrade" alongside existing tab buttons
- When Upgrade tab is active, show the player's current deck cards as a list
- Each card shows: card name (with + if already upgraded), card rarity, upgrade gold cost (from `ShopSystem.getUpgradePrice(card.rarity)`), and "UPGRADED" label if already upgraded
- Cards already in `getRun().deck.upgradedCards` are shown grayed out with "UPGRADED" text
- Clicking an upgradeable card calls `ShopSystem.upgradeCard(runState, cardId, card.rarity)` using the actual RunState (not the local interface)

- Use the same Phaser text/button patterns as existing tabs (match font size, colors, positioning)

**2. src/scenes/CombatScene.ts** -- Add visual distinction for upgraded cards.

Read the existing card queue rendering code. Where cards are displayed in the combat UI (card name text objects):

- When rendering a card name in the queue, check if `getRun().deck.upgradedCards.includes(cardId)`
- If upgraded, append "+" to the card name display (e.g., "Strike+")
- Optionally: set the card name text color to gold (0xFFD700) for upgraded cards to provide visual distinction
- This is display-only -- no gameplay logic changes needed here (CardResolver already handles the actual effects)

**3. src/scenes/EventScene.ts** -- Handle upgrade_card effect from events.

Read the existing EventScene to understand how event choice effects are displayed. When an event choice includes `upgrade_card` effect:

- After `resolveEventChoice` returns with an `upgrade_card` effect where `applied: true`, pick a random non-upgraded card from `getRun().deck.active` that is NOT in `getRun().deck.upgradedCards`
- Add that card ID to `getRun().deck.upgradedCards`
- Display the upgrade result to the player (e.g., "Strike was upgraded to Strike+!")
- If all cards are already upgraded, display "No cards to upgrade"
